This allows Visual Studio to automatically detect and use libraries installed via vcpkg.\hypertarget{md__r_e_a_d_m_e_autotoc_md6}{}\doxysubsection{\texorpdfstring{Step 3\+: Build the project}{Step 3\+: Build the project}}\label{md__r_e_a_d_m_e_autotoc_md6}
vcpkg should automatically download and install the required dependencies while building the project in Visual Studio, based on the {\ttfamily vcpkg.\+json} manifest file.\hypertarget{md__r_e_a_d_m_e_autotoc_md7}{}\doxysubsection{\texorpdfstring{Step 4\+: Not visible opencv2 workaround}{Step 4\+: Not visible opencv2 workaround}}\label{md__r_e_a_d_m_e_autotoc_md7}
If the paths to the Open\+CV libraries are not visible in Visual Studio, you should go to the {\ttfamily vcpkg\+\_\+installed/x64-\/windows/x64-\/windows/include/opencv4} folder and move the {\ttfamily opencv2} folder outside {\ttfamily opencv4}, to {\ttfamily vcpkg\+\_\+installed/x64-\/windows/x64-\/windows/include} folder. This is a workaround for the issue where Open\+CV libraries are not detected properly in Visual Studio because the {\ttfamily opencv2} folder is nested inside {\ttfamily opencv4}. This should make the Open\+CV libraries visible in Visual Studio. You can also remove the {\ttfamily opencv4} folder if you want to keep the structure clean.

Side note\+: The best practice is to keep the {\ttfamily opencv2} folder inside {\ttfamily opencv4} and use the correct include paths in your project settings, but this workaround is a quick fix for the issue and i couldn\textquotesingle{}t find a better working solution.
